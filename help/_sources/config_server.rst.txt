Секция Server
=========================================
Здесь задаются параметры использования :doc:`REST API <api>`:

.. contents:: :local:


Host
----------------------------------------
Задаёт адрес на котором приложение будет ожидать команды.

Можно использовать как IP адрес, так и hostname. 

.. code-block:: cfg

	host=localost
	host=172.22.16.54

.. warning::

	Лутника будет принимать команды только по заданному адресу.


Port
----------------------------------------
Задаёт порт на котором он будет ожидать команды.

Этот порт используется как для управления приложением, так и для управления заданиями.

.. code-block:: cfg

	port=7252


QueryClients
----------------------------------------
Список адресов для которых разрешено выполнять :doc:`api_common`.

Можно использовать как IP адрес, так hostname или укажите :guilabel:`*` чтобы дать разрешения всем.
Разделитель :guilabel:`;` используется для перечисления нескольких адресов.

.. code-block:: cfg

	QueryClients=*
	QueryClients=localhost; 172.22.16.54;myserver

.. warning::
	Все входящие команды от клиентов вне этого списка будут отклонены.

AdminClients
----------------------------------------
Список адресов для которых разрешено выполнять :doc:`api_control`.

Указанные здесь клиенты уже имеют уровень доступа :option:`QueryClients` и добавлять их туда не имеет смыла. Указывайте здесь только те адреса, с которых необходимо управлять приложением.

Можно использовать как IP адрес, так hostname или укажите :guilabel:`*` чтобы дать разрешения всем.
Разделитель :guilabel:`;` используется для перечисления нескольких адресов.

.. code-block:: cfg

	AdminClients=*
	AdminClients=localhost; 172.22.16.54;myserver

.. warning::
	Все входящие команды от клиентов вне этого списка будут отклонены.
